# 作業驗收

{% hint style="info" %}
作業類型

1. 個人作業：課堂競賽之盾的作業類型以外的作業皆屬於此項
2. 共管教師作業：班級內其他教師（共管教師）所出的作業
3. 世界作業：官方所派發的世界任務，其作業則屬於此項
4. 競賽之盾：若將作業分派為課堂競賽之盾，其作業屬於此項
{% endhint %}

### 如何查看班級學生於單一作業下的作答標準，檢視其對應的人數？

<mark style="background-color:purple;">檢視位置：教師後台 -> 統計分析 -> 作業驗收</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-07-19 11.24.04.png" alt="" width="563"><figcaption></figcaption></figure>

教師選定班級與作業後，能夠滑動完成度與作業分數的滑桿，取得一定標準下對應的人數。

1. 完成度：以「個別學生的任務完成度」作為篩選標準
2. 作業分數：以「個別學生的作業分數」作為篩選標準\
   （作業分數＝作業完成度＊作業正確率＊100）

{% hint style="info" %}
如果某一個任務同時指派到兩個班級，而作答的用戶同時存在於這兩個班級中，系統會把作答資料紀錄給「先建立的那個班級」
{% endhint %}

### 如何查看班級學生於單一作業內答錯率最高的題目？

<mark style="background-color:purple;">檢視位置：教師後台 -> 統計分析 -> 作業驗收 -> 錯題排行榜</mark>

教師選定班級與作業後，能夠查看該作業內答錯次數較多的題目。另外，教師也可以點擊錯題排行榜右下方的「一鍵出題」，將錯題打包成新作業再派發任務給學生。

{% hint style="warning" %}
如果錯題排行榜中並無出現一鍵出題的按鈕，原因是系統不開放教師使用這項功能，詳情參考[此連結](../../admin-courses/game-setting/jiao-shi-hou-tai.md#ru-he-da-kai-jiao-shi-hou-tai-de-yi-jian-chu-ti-de-gong-neng)
{% endhint %}

### 如何一鍵打包單一作業的錯題排行榜的錯題？

<mark style="background-color:purple;">設定位置：教師後台 -> 統計分析 -> 作業驗收 -> 錯題排行榜</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-11-01 15.50.36.png" alt="" width="563"><figcaption></figcaption></figure>

1. 選擇欲派發錯題排行榜的班級與作業
2. 於錯題排行榜的區塊中點擊「一鍵出題」
3. 依照流程新增作業與任務

### 如何批改申論題？

<mark style="background-color:purple;">設定位置：教師後台 -> 統計分析 -> 作業驗收 -> 申論題</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-11-01 16.00.35.png" alt="" width="563"><figcaption></figcaption></figure>

若該作業內有包含申論題的題目，申論題的作答狀況（完成與未完成的人數）會被獨立出來。

1. 選擇欲批改申論題的的班級與作業
2. 於申論題的區塊中點擊欲批改題目旁的「批閱」
3. 勾選欲批改申論題內容的學生，並於下方欄位輸入批改內容
4. 點擊「送出」後，該則批改內容即會發送到作答學生的個人訊息中

測驗模式下，有兩種用戶未作答的情境，在批閱介面上會顯示固定的提示文字：

1. 未作答就直接送出，則介面上會顯示「NO\_ANSWERS」
2. 未作答但測驗逾時，則介面上會顯示「TIMEOUT」

{% hint style="info" %}
若是一般任務中的申論題，上面兩種情境會被視為答錯而不會寫入批閱後台。且因為答錯所以用戶可以再次作答，當用戶有作答後，答案才會出現在批閱後台中。
{% endhint %}

### 如何查看班級學生於單一作業的正確率、完成率、作業分數？

<mark style="background-color:purple;">設定位置：教師後台 -> 統計分析 -> 作業驗收 -> 學生列表</mark>

教師選定班級與作業後，能夠查看該班級內的學生於此單一作業的答題狀況。

（數據皆為即時數據，可以立即反應當下完成的題目數量）

1. 正確率：以「作業正確率」作為數據來源：作業正確率加總 / 班級總人數\
   多選
2. 完成度：以「作業完成度」作為數據來源：該作業完成人數 / 班級總人數\
   無論何種題目類型只要送出，皆算是完成該作業
3. 作業分數：作業完成度＊作業正確率＊100

{% hint style="info" %}
如果某一個任務同時指派到兩個班級，而作答的用戶同時存在於這兩個班級中，系統會把作答資料紀錄給「先建立的那個班級」
{% endhint %}

### 如何查看學生於不同作業下的正確率、完成率、作業分數？

<mark style="background-color:purple;">設定位置：教師後台 -> 統計分析 -> 作業驗收 -> 章節分析</mark>

作業裡可能會包含多種不同章節，教師選定班級與作業後，能夠查看該作業裡不同章節的個別作答狀況

1. 章節正確率：以章節內題目的「正確率」作為數據來源
2. 章節完成度：以章節內題目的「完成度」作為數據來源
3. 章節分數：章節完成度＊章節正確率＊100^2
