# 使用者帳號管理

{% hint style="info" %}
1. 每一個使用者僅會有一個帳號，但在不同世界中可分別建立不同的遊戲角色
2. 每一個使用者可能有多個遊戲角色，其數目由該使用者的參與課程數量而定
{% endhint %}

### 如何查看或編輯使用者的資訊？

<mark style="background-color:purple;">設定位置：管理者後台 -> 系統設定 -> 使用者帳號管理</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-17 下午2.36.36.png" alt="" width="563"><figcaption></figcaption></figure>

1. 可透過 UID、帳號、真實姓名、暱稱、email、教育雲 email，精準搜尋特定的使用者
2. 可於列表中查看使用者的 UID、帳號、真實姓名、暱稱、email、已授權的第三方登入、教育雲 email
3. 如需要進一步查看或編輯使用者的資訊，須點擊「查看」
   * 除列表上的資訊外，還可查看大頭貼、第三方 email、手機號碼、參與課程、管理者權限、副管理者權限、開設課程權限、批量上傳權限（具備管理者權限的帳號，才可看到管理者權限＆副管理者權限開關）
   * 可點擊「編輯」更改真實姓名、暱稱、email、管理者權限、副管理者權限、開設課程權限、批量上傳權限

{% hint style="info" %}
* 管理者權限：能夠取用管理者後台的所有功能，也能夠設定其他人為管理者或是副管理者
* 副管理者權限：能夠取用管理者後台的所有功能
* 開設課程權限：開設課程權限。另外，無設定管理者與副管理者權限，此設定值才有效。e.g. 如果Ａ同仁持有管理者與副管理者權限，但是沒有開啟課程權限，則Ａ同仁還是能夠開設課程
* 批量上傳權限：若取得批量上傳權限，該使用者可於教師後台的個人題庫管理中批量上傳題
{% endhint %}

### 如何移除大頭貼？

<mark style="background-color:purple;">設定位置：管理者後台 -> 系統設定 -> 使用者帳號管理 -> 查看 -> 編輯</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-17 下午3.13.59.png" alt="" width="563"><figcaption></figcaption></figure>

進入「編輯」畫面後，若使用者有上傳大頭貼，則可以點擊頭貼旁的「移除 icon」，進而移除使用者的大頭貼

### 如何寄重設密碼信？

<mark style="background-color:purple;">設定位置：管理者後台 -> 系統設定 -> 使用者帳號管理</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-17 下午3.09.43.png" alt="" width="563"><figcaption></figcaption></figure>

點擊「操作」欄位的「寄重設密碼信」按鈕，進而寄送重設密碼的信件

{% hint style="info" %}
* 點擊「寄重設密碼信」後的 60 秒內，無法再次寄送，故按鈕會 disabled
* 若是該帳號尚未完成驗證，按鈕也會 disable
{% endhint %}

### 如何寄送驗證信/直接驗證？

<mark style="background-color:purple;">設定位置：管理者後台 -> 系統設定 -> 使用者帳號管理</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-17 下午2.51.04.png" alt="" width="563"><figcaption></figcaption></figure>

滑鼠 hover 到「操作」欄位的「⋯」按鈕後，點擊「寄送驗證信」或「直接驗證」按鈕，進而寄送驗證帳號的信件，或是幫使用者直接完成驗證

{% hint style="info" %}
* 點擊「寄送驗證信」後的 60 秒內，無法再次寄送，故按鈕會 disabled
* 若是該帳號已完成驗證，則「寄送驗證信」和「直接驗證」按鈕皆會 disabled
{% endhint %}

### 如何凍結 / 解凍帳號？

<mark style="background-color:purple;">設定位置：管理者後台 -> 系統設定 -> 使用者帳號管理</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-17 下午2.53.49.png" alt="" width="563"><figcaption></figcaption></figure>

1. 滑鼠 hover 到「操作」欄位的「⋯」按鈕後，點擊「凍結」按鈕，進而將使用者凍結，讓使用者無法登入帳號使用
2. 若使用者已被凍結中，則可以點擊「解凍」按鈕，進而將使用者解凍，恢復帳號的使用權
