# 創建訂閱商品帳號

### 如何創建訂閱商品帳號？

<mark style="background-color:purple;">設定位置：管理者後台 -> 課程世界設定 -> 批量創建設定 -> 訂閱商品帳號創建</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-09 13.44.17.png" alt="" width="563"><figcaption></figcaption></figure>

在創建訂閱商品帳號時，可選擇**單獨創建**老師或是學生帳號（也開放同時創建兩種帳號）。老師帳號可選擇不綁定商品教材；而學生帳號則至少需綁定一商品教材。

1. 下載範例檔案並依照欄位要求填上資料
2. 上傳「學生檔案」與「教師檔案」
3. 點選「上傳並創建帳號」
4. 於下方列表檢視匯出資料

{% hint style="info" %}
1) 範例檔案中，標題欄位名稱皆需保留
2) 標題有 \* 的欄位為必填欄位
3) 每位學生或師長的學號、信箱不可重複
4) 若學生密碼為空，則預設為與學生帳號相同
5) &#x20;如有設置密碼，僅能是大小寫英文+數字，不能還有特殊字元，最少八個字元
6) 若有填寫學生信箱，學生登入時可使用信箱或帳號作為帳號登入的兩種方式
7) 綁定教材的欄位需要於表頭填欲綁定的教材 Slug，並於其下方的欄位中填入該任務的「結束派發日」
8) 若教師綁定商品教材，任務起派日為必填
{% endhint %}

{% hint style="info" %}
帳號整併方式

1. 留下舊的帳號，把新的帳號砍掉
2. 新帳號中有 RD 寫的東西會移轉過去，除了不同世界的內容外，目前已知還有訂閱制商品會移轉
3. 由於會搬移世界中的物品，所以在同世界中有角色的話不能合併
{% endhint %}

{% hint style="warning" %}
在 2S / 2G 的情境下，通常不會開放整併帳號的功能給學生，因為其機制為留下舊帳號，將新的帳號移除，如果學生以舊帳號取代之，則營運還需要另外更新帳號名單
{% endhint %}
