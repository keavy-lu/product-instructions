# 創建一般帳號

### 如何創建一般帳號？

<mark style="background-color:purple;">設定位置：管理者後台 -> 課程世界設定 -> 批量創建設定 -> 創建一般帳號</mark>

<figure><img src="../../../.gitbook/assets/截圖 2023-08-09 13.44.10.png" alt=""><figcaption></figcaption></figure>

在創建一般帳號時，其所創建出來的帳號都會是學生帳號，且不會綁定商品教材

1. 輸入欲創建的帳號數量（1\~300），並點擊「批量創帳號」
2. 進入初始設定頁面，並完成以下設定
   * 設定暱稱真實姓名是否一致
   * 設定真實姓名前綴[^1]
   * 選擇密碼設定
   * 設定初始金錢
   * 設定初始 QP（原則上學生帳號並不會使用到QP，若該帳號之後獲得教師後台權限後，會直接獲得該初始 QP）
   * 設定帳號到期日
   * 設定帳號是否整併
   * 選擇角色設定
   * 選擇該次批量創建時，帳號需匯入的班級（須為現有班級）
3. 進入帳號設定頁面，並完成以下設定（皆選填）
   * 設定真實姓名
   * 設定手機號碼
   * 設定電子信箱

{% hint style="info" %}
帳號整併方式

1. 留下舊的帳號，把新的帳號砍掉
2. 新帳號中有 RD 寫的東西會移轉過去，除了不同世界的內容外，目前已知還有訂閱制商品會移轉
3. 由於會搬移世界中的物品，所以在同世界中有角色的話不能合併
{% endhint %}

{% hint style="warning" %}
如果設定真實姓名前綴，會一併把課程代碼改掉改掉，請謹慎使用
{% endhint %}

{% hint style="warning" %}
在 2S / 2G 的情境下，通常不會開放整併帳號的功能給學生，因為其機制為留下舊帳號，將新的帳號移除，如果學生以舊帳號取代之，則營運還需要另外更新帳號名單
{% endhint %}

[^1]: 若在此處設定，則「帳號設定」tab 裡的真實姓姓名將即時帶入前綴
